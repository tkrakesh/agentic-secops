# PB-003: Credential Compromise Response

## Playbook Metadata
- **Playbook ID**: PB-003
- **Version**: 2.1
- **Last Updated**: 2026-01-15
- **Owner**: Identity Security Team
- **SLA**: Containment within 60 minutes of alert
- **Approval Required**: Security Manager (automated pre-approval for CRITICAL severity)

## Trigger Conditions
This playbook activates when ALL of the following conditions are met:
1. Lateral movement alerts detected (sequential host access pattern ≥3 hosts within 30 minutes)
2. Valid domain account exhibiting anomalous behaviour (off-hours access, unusual source IP, multiple simultaneous sessions)
3. Active Directory audit log anomalies (NTLM authentication, explicit credential logon events 4648/4624)
4. EDR telemetry confirms remote execution tool usage (PsExec, WMI, RDP chaining)

**MITRE ATT&CK Coverage**: T1078 (Valid Accounts), T1021.002 (Remote Services: SMB/Windows Admin Shares), T1569.002 (PsExec), T1550.002 (Pass the Hash)

## Action Sequence

### Step 1 — Disable Compromised Account (Automated)
- Immediately disable the identified user account in Active Directory
- Invalidate all active Kerberos tickets (klist purge equivalent via API)
- Force MFA re-enrolment on next login
- **Target**: All accounts exhibiting lateral movement behaviour
- **Tool**: AD API / IAM platform
- **Expected Duration**: 2 minutes

### Step 2 — Force Password Reset
- Generate temporary password and trigger mandatory reset flow
- Notify user's manager and IT Service Desk via automated email
- Log reason: "Emergency security suspension — investigation in progress"
- **Expected Duration**: 1 minute

### Step 3 — Isolate Source Endpoint
- Apply network isolation policy to the originating workstation
- Allow only SOC investigation traffic (port 443 to SIEM, 22 to jump server)
- Block all outbound internet and lateral network communication
- **Tool**: EDR (CrowdStrike) network containment API
- **Expected Duration**: 2–3 minutes

### Step 4 — Collect Active Directory Audit Logs
- Pull last 72 hours of AD audit events for the compromised account
- Export to secure evidence bucket (read-only, hash verified)
- Include: 4624, 4625, 4648, 4768, 4769, 7045 event IDs
- **Expected Duration**: 5 minutes

### Step 5 — Notify Identity & Access Management Team
- Send P1 notification to IAM On-Call engineer
- Include: affected account, compromised hosts, timeline, AD export reference
- Open bridge call if ≥5 hosts affected
- **Expected Duration**: 1 minute

## Estimated Containment Time
**12–15 minutes** (automated steps 1–3: ~5 minutes, evidence collection: 5 minutes, notification: 1 minute)

## Post-Containment Actions
- Forensic analysis of DESKTOP-DB42 (likely patient zero)
- Review AD group membership for privilege escalation indicators
- Check for new accounts or group membership changes in 72h window
- Threat hunt for persistence mechanisms (scheduled tasks, registry run keys, startup items)

## Exclusion Criteria
- Do NOT activate if alert source is a known automated patching account
- Do NOT activate if source IP matches approved IT admin VPN range (check CMDB)
- Escalate to Incident Commander if ≥10 hosts affected
