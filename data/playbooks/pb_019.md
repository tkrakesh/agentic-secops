# PB-019: Phishing Response

## Playbook Metadata
- **Playbook ID**: PB-019
- **Version**: 2.4
- **Last Updated**: 2025-12-10
- **Owner**: Email Security Team
- **SLA**: Domain blocked within 20 minutes; mailbox quarantine within 45 minutes
- **Approval Required**: Security Analyst (L2) or above

## Trigger Conditions
1. User reports suspicious email AND link click confirmed in proxy logs
2. Credential harvest page loaded (Microsoft-lookalike, banking portal spoofs, MFA fatigue pages)
3. Authentication from unusual geography or impossible travel detection within 30 minutes of click
4. Confirmed malicious URL in email from threat intelligence lookup (reputation ≥70)

**MITRE ATT&CK Coverage**: T1566.001 (Spearphishing Attachment), T1056.003 (Web Portal Capture), T1110.003 (Password Spraying post-harvest), T1078 (Valid Accounts)

## Action Sequence

### Step 1 — Block Sender Domain
- Add sender domain to email gateway deny list
- Block domain at DNS and proxy layers
- Search for and quarantine all emails with same sender/domain across all mailboxes
- **Tool**: Email Security Gateway API
- **Expected Duration**: 5 minutes

### Step 2 — Quarantine Phishing Email Across All Mailboxes
- Pull message ID and quarantine across all mailboxes (use Microsoft Defender / Gmail Admin API)
- Document: sender, subject, recipients, timestamp, links, attachments
- **Expected Duration**: 10 minutes

### Step 3 — Disable Session Tokens for Victim Accounts
- Revoke all active OAuth tokens and session cookies for affected users
- Force reauthentication at next access
- **Expected Duration**: 3 minutes

### Step 4 — Force MFA Re-Enrolment
- Reset authenticator app binding
- Require re-enrolment with verified phone number
- Alert user via SMS (not email, which may be compromised)
- **Expected Duration**: 5 minutes

### Step 5 — User Awareness Notification
- Send personalised awareness notice to victim user explaining what happened
- Include guidance on what to report next (unusual activity, new device logins)
- Escalate to mandatory security awareness training module
- **Expected Duration**: 2 minutes

## Estimated Containment Time
**25–30 minutes** for full containment. Phishing campaign investigation: 2–4 hours.

## Post-Containment Actions
- Full analysis of harvest page (screenshot, network captures)
- Check for credential reuse across other systems
- Investigate whether MFA was bypassed (AiTM attack indicators)
- Threat hunt for other recipients of same campaign

## Exclusion Criteria
- False positive: legitimate marketing email from unfamiliar domain
- Escalate to PB-003 if credential harvest succeeded and lateral movement is detected
