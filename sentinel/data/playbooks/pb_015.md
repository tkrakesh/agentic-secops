# PB-015: Policy Compliance & Certificate Refresh (TLS/SSL)

## Playbook Metadata
- **Playbook ID**: PB-015
- **Version**: 1.0
- **Severity**: LOW
- **SLA**: 24 hours
- **Approval Required**: Auto-approved for internal assets.

## Action Sequence
1. **Identify Protocol**: Confirm the deprecated TLS/SSL version or insecure cipher suite being negotiated.
2. **Scan Stakeholders**: Identify all services currently violating the security policy or compliance requirements.
3. **Notify Admins**: Send an automated compliance warning to the system owners regarding the insecure configuration.
4. **Trigger Update**: Initiate automated certificate refresh or configuration update via Ansible/Terraform to enforce TLS 1.2+ compliance.

## Containment Logic
This playbook focuses on remediation of configuration drift and policy violations. No network isolation is required unless an active exploit is suspected.
