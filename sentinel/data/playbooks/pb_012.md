# PB-012: Ransomware Isolation Protocol

## Playbook Metadata
- **Playbook ID**: PB-012
- **Version**: 4.2
- **Last Updated**: 2026-02-01
- **Owner**: Critical Incident Response Team (CIRT)
- **SLA**: All affected endpoints isolated within 15 minutes of confirmation
- **Approval Required**: CISO or Deputy CISO (or SOC Manager on-call after hours)
- **Priority**: P0 — Maximum Urgency

## Trigger Conditions
This playbook activates when ALL of the following are detected:
1. Encoded or obfuscated PowerShell execution on ≥2 endpoints (T1059.001 with T1027.001 AMSI bypass)
2. Memory-resident malware signature confirmed by EDR (Cobalt Strike, Metasploit, PowerShell Empire)
3. Lateral spread pattern within 30 minutes (≥2 endpoints sharing same beacon/stager signature)
4. C2 beacon heartbeat confirmed to known malicious IP or domain

**MITRE ATT&CK Coverage**: T1059.001 (PowerShell), T1055 (Process Injection), T1027 (Obfuscated Files), T1486 (Data Encrypted for Impact — anticipated), T1490 (Inhibit System Recovery — anticipated)

## ⚠ CRITICAL: Pre-Execution Checklist
Before executing this playbook, confirm:
- [ ] Incident Commander notified and on bridge call
- [ ] Backup systems verified NOT connected to affected segment
- [ ] Executive stakeholders (CISO, CTO) aware if >10 hosts affected
- [ ] Legal and Communications team notified if customer data at risk

## Action Sequence

### Step 1 — Emergency Endpoint Isolation (ALL affected hosts) — IMMEDIATE
- Apply CrowdStrike network containment to ALL affected endpoints simultaneously
- Do NOT isolate sequentially — ransomware deployment may accelerate on alert detection
- Block ALL network traffic except SIEM telemetry
- **Tool**: CrowdStrike Falcon bulk containment API
- **Expected Duration**: 2–5 minutes

### Step 2 — Disable Affected User Accounts
- Disable all user accounts logged into affected endpoints
- Force-expire all active sessions and Kerberos tickets
- Do NOT alert the users — ransomware affiliates often monitor user behaviour
- **Expected Duration**: 3 minutes

### Step 3 — Snapshot Affected Systems (Forensic Preservation)
- Trigger VM snapshots or EDR memory dumps on all affected endpoints before any remediation
- Upload to secure, isolated forensic storage (separate network segment, read-only)
- Record hash of each snapshot immediately
- **Expected Duration**: 10–15 minutes

### Step 4 — Block C2 IPs and Domains at Perimeter
- Add all confirmed C2 IPs and domains to emergency deny rules on all perimeter firewalls
- Apply across ALL firewall zones — not just the affected VLAN
- Check for DNS-over-HTTPS bypass paths and block
- **Expected Duration**: 5 minutes

### Step 5 — Initiate Incident Bridge
- Open P0 incident bridge with CIRT, Network Security, Identity, and Business Continuity teams
- Declare Major Incident (MI) in ServiceNow
- Brief Incident Commander on blast radius, timeline, and actions taken
- **Expected Duration**: Immediate — parallel to steps 1–4

## Estimated Containment Time
**15–20 minutes** to full network isolation. Full remediation (re-imaging, restore) 4–72 hours depending on scope.

## Post-Containment Actions
- Full malware reverse engineering of Cobalt Strike payload
- Identify initial access vector (phishing, vulnerability exploitation, supply chain)
- Restore from last known-good backup (verify backup integrity before restore)
- Strategic threat hunt across all endpoints for persistence mechanisms
- Regulatory notification assessment (DORA, PRA, ICO within 72h of discovery)

## Exclusion Criteria
- Do NOT delay isolation pending management approval — time-critical
- If >50 hosts affected, invoke Business Continuity Plan alongside this playbook
