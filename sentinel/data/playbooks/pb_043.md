# PB-043: Ransomware Staging & Tooling Defense

## Playbook Metadata
- **Playbook ID**: PB-043
- **Version**: 1.0
- **Severity**: CRITICAL
- **SLA**: 30 minutes to isolation
- **MITRE ATT&CK**: T1110 (Brute Force), T1140 (Deobfuscate/Decode), T1105 (Ingress Tool Transfer), T1059.001 (PowerShell)

## Trigger Conditions
1. Multiple failed login attempts (brute force) followed by a successful login.
2. Use of `certutil.exe` with `-urlcache` or `-decode` flags for suspicious file downloads (e.g., .js, .exe).
3. Execution of known ransomware precursor tools (e.g., `AgentTesla.js`, `Gift4You.exe`).

## Action Sequence

### Step 1 — Immediate Endpoint Isolation
- Quarantine the host using EDR to prevent lateral movement and C2 beaconing.
- **Tool**: CrowdStrike Falcon / SentinelOne Isolation.

### Step 2 — Account Lockdown
- Disable the compromised account (e.g., ADMINZERO).
- Invalidate all active sessions for the account.
- **Tool**: Active Directory / Azure AD.

### Step 3 — Process Termination & Malware Removal
- Kill the malicious processes and move identified files to the EDR quarantine vault.
- **Tool**: EDR Remediation.

### Step 4 — Threat Sweep
- Scan all other endpoints for matching hashes or similar brute force patterns from the same source IP.
- **Tool**: EDR Sweep.

## Containment Logic
If brute force leads to a successful login followed by `certutil` downloading a file, high-confidence ransomware staging is assumed, and immediate isolation is mandatory.
