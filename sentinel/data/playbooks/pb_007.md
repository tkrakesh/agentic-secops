# PB-007: C2 Containment & Forensics

## Playbook Metadata
- **Playbook ID**: PB-007
- **Version**: 3.0
- **Last Updated**: 2025-11-20
- **Owner**: Network Security Team
- **SLA**: C2 domain blocked within 30 minutes; full forensic capture within 2 hours
- **Approval Required**: Security Manager

## Trigger Conditions
This playbook activates when ANY of the following are confirmed:
1. DNS tunnelling detected (query rate >500/min to non-whitelist domain OR TXT record data exfiltration signatures)
2. Known C2 domain or IP match in threat intelligence with reputation score ≥80
3. Large outbound data transfer (>500 MB) via non-standard protocol or DLP bypass detected
4. Proxy bypass indicators combined with known C2 infrastructure contact

**MITRE ATT&CK Coverage**: T1071.004 (DNS), T1048.003 (Exfil over Alternative Protocol), T1041 (Exfiltration over C2), T1102 (Web Service C2)

## Action Sequence

### Step 1 — Block C2 Domain at DNS Layer (Automated)
- Add domain(s) to DNS sinkhole / Response Policy Zone (RPZ)
- Block at all internal DNS resolvers (primary, secondary, forwarders)
- Add to firewall FQDN deny list
- **Tool**: DNS Firewall / Palo Alto FQDN object
- **Expected Duration**: 3 minutes

### Step 2 — Block Egress IP at Perimeter Firewall
- Create deny rule for all identified C2 IP addresses on perimeter and core firewalls
- Apply to both outbound HTTPS (443) and DNS UDP/TCP (53) rule sets
- Log and alert on any bypass attempts
- **Tool**: Palo Alto Firewall API / Security Policy Manager
- **Expected Duration**: 5 minutes

### Step 3 — Isolate Affected Endpoints
- Apply network quarantine to all endpoints with confirmed C2 communication
- Allow SIEM telemetry, DNS resolution to internal domains only
- **Tool**: CrowdStrike Falcon network containment
- **Expected Duration**: 3–5 minutes per endpoint

### Step 4 — Capture Network Forensics
- Enable full packet capture (PCAP) on the affected network segment for 30 minutes
- Pull DNS server logs for the previous 48 hours
- Export proxy logs for affected source IPs
- Hash all evidence files and store in secure SIEM artifact store
- **Expected Duration**: 30 minutes

### Step 5 — PCAP Collection and DLP Assessment
- Collect and decrypt (where possible) PCAP from firewall
- Identify data types transferred (structured data, credentials, documents)
- Estimate volume and sensitivity of exfiltrated data
- Notify Data Protection Officer if PII/financial data is confirmed
- **Expected Duration**: 45 minutes

## Estimated Containment Time
**30–45 minutes** (network blocks: 10 minutes, endpoint isolation: 15 minutes, forensic setup: 20 minutes)

## Post-Containment Actions
- IOC sharing with threat intelligence feeds
- Full forensic analysis of source endpoint
- Confirm data at rest encryption status for any staging directories
- Review DLP rule coverage gaps that allowed bypass
- Notify compliance team if financial/customer data confirmed exfiltrated

## Exclusion Criteria
- Do NOT activate for false positive DNS anomalies from approved monitoring tools
- Confirm C2 domain is not a legitimate misconfigured internal service
